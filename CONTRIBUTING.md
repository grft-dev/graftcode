# Contributing to Graftcode

Thank you for considering contributing to Graftcode! ❤️

## How to Contribute

We welcome contributions in many forms:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Submitting fixes and improvements (especially to plugins and demos)
- Creating examples and integrations

### Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch (`git checkout -b feature/amazing-thing`)
4. Make your changes
5. Test them thoroughly
6. Submit a Pull Request

### Pull Request Guidelines

- Keep changes focused on a single feature or fix
- Update documentation if needed
- Follow existing code style
- Add tests where applicable
- Make sure all existing tests pass

### Communication

- For questions and discussions → [Discord](https://discord.gg/2tWb3BAE36)
- For bugs and feature requests → use GitHub Issues

---

**Note**: The core of Graftcode is currently closed-source, but we are actively open-sourcing more components (plugins, tools, examples). Contributions to open-source parts are especially welcome.
